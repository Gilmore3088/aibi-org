## Issue #92 — homepage ladder + funnel QA

### Bug 92-1 — pre-email score CTA hardcoded to Foundation for lower tiers
**Found:** Playwright drove the free 12-question assessment to a `starting-point` score; the big primary CTA on the score-reveal screen read "Explore the Foundation course" and routed to `/courses/foundation/program`.

**Root cause:** `src/app/assessment/page.tsx` line 94-100. `isLowerTier` branch hardcoded a Foundation CTA. `NextStepCards` correctly routes lower tiers to In-Depth as primary, but `NextStepCards` only renders POST email capture, so the pre-email funnel never reached it.

**Per issue #88/#92:** lower tiers (`starting-point`, `early-stage`) should land on the In-Depth Assessment ($99) as the deeper diagnostic before the $295 course commitment.

**Fix (commit `5792e17`):** swap the lower-tier pre-email CTA to "Take the In-Depth Assessment · $99" → `/assessment/in-depth`. Higher tiers continue to Executive Briefing.

**Re-verified:**
- starting-point → "Take the In-Depth Assessment · $99" ✓
- early-stage → "Take the In-Depth Assessment · $99" ✓
- building-momentum → "Book Your Executive Briefing" ✓
- ready-to-scale → "Book Your Executive Briefing" ✓


### Bug 92-2 — /assessment had no H1
**Found:** Axe-style a11y pass on `/assessment` returned headings `H2,H3` only. No H1 means screen-reader users have no top-level page anchor.

**Fix:** add `<h1 className="sr-only">AI Readiness Assessment</h1>` at the top of the `<main>` in `src/app/assessment/page.tsx`. Visually hidden so layout doesn't shift; semantically present.

**Re-verified:** `/assessment` now reports `H1,H2,H3`.

### Bug 92-3 — Foundation pages skipped heading levels (H1 → H3)
**Found:** `/courses/foundation/program` and `/courses/foundation/program/purchase` rendered `H1, H3, H2, H3` — H3 appearing before H2 violates WCAG 1.3.1.

**Root cause:** In both pages the "current module name" / "$295 price" block used `<h3>` as a visual style, not a semantic level.

**Fix:** bump `<h3>` → `<h2>` for the current-module title and the price block. Same visual; correct hierarchy.

**Re-verified:** both pages now report `H1, H2, H2, H3`.

### Bug 92-5 — `/robots.txt` and `/sitemap.xml` get rewritten to coming-soon HTML
**Found:** Playwright smoke suite failed `robots.txt disallows /api, /auth, /dashboard`. Fetched the route manually — it returned `text/html` with the "We will be helping you with all the AI noise soon" coming-soon body instead of the robots.txt rules.

**Root cause:** `src/middleware.ts` only bypassed prefix paths (`/api`, `/assessment`, …) under `COMING_SOON=true`. `/robots.txt` and `/sitemap.xml` aren't in any prefix and got rewritten. Same issue applies to the favicon SVGs.

**Why this matters in production:** if the site goes into coming-soon mode again (e.g., during a maintenance window), search-engine crawlers will see HTML at `/robots.txt` and `/sitemap.xml` instead of the actual SEO rules — degrades indexing for the post-takedown launch.

**Fix:** added `COMING_SOON_BYPASS_EXACT` list with `/robots.txt`, `/sitemap.xml`, `/icon.svg`, `/apple-icon.svg`. Middleware bypass logic checks exact-match list before the prefix list.

**Re-verified:** `/robots.txt` now serves plain-text rules, `/sitemap.xml` serves XML.

### Bug 92-6 — homepage tagline e2e test fails on strict-mode (two valid matches)
**Found:** smoke test `homepage renders with tagline` failed because `getByText(/Turning Bankers into Builders/i)` matched both the hero H1 and the SiteFooter wordmark subtitle (two legitimate places — brand reinforcement).

**Fix:** scope the assertion to the hero H1: `page.getByRole('heading', { level: 1, name: /Turning Bankers into Builders/i })`. Test-side change only.

### Bug 92-7 — 404 e2e test fails under COMING_SOON takedown
**Found:** smoke test `404 page renders for unknown route` failed with `status=200` because unknown routes get rewritten to `/coming-soon` while takedown is active (correct production behavior).

**Fix:** detect the coming-soon body in the response and `test.skip(true, …)` so the assertion only fires when the real site is live. Test-side change only.
