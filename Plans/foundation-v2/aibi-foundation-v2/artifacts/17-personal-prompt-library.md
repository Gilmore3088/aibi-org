# Personal Prompt Library

**Module 11 artifact · AiBI-Foundation · Spine artifact**

This library is the durable record of what works in your AI use. It is also the input format for AiBI-Specialist's Departmental Skill Library — **the schema below is fixed**. Don't change field names.

**Learner:** _______________________________________________
**Role:** _______________________________________________
**Department:** _______________________________________________
**Library created:** _______________
**Total entries:** ___
**Next quarterly review:** _______________

---

## Library schema (use this for every entry)

| Field | Description |
|---|---|
| ID | Short reference, 3 letters + 2 digits (e.g., MEM-01, LND-02) |
| Name | What this prompt does (one short phrase) |
| Task type | Drafting / summarizing / extracting / format-shifting / Q&A / workflow |
| Role | Who uses this |
| Frequency | How often it runs |
| Data tier | Public / Internal / Confidential / Restricted |
| Tool tier | Public AI / Copilot Chat / M365 Copilot / Approved specialist |
| Tool | Specific tool |
| Project | Linked project name from Module 6, if applicable |
| System prompt | Persistent instructions, paste verbatim |
| Sample input prompt | Example of how to invoke it |
| Sample output | Example of good output |
| Verification step | How to check the output |
| Pre-flight check | Reference to Module 3 (default: "All five questions") |
| Time saved per use | Honest estimate |
| Last tested | Date last verified working |
| Quarterly review | Next scheduled review |
| Notes / what didn't work | Lessons from refinement |

## Maintenance plan

### Review cadence

- [ ] Quarterly (default) — calendar reminder set on: ___________________
- [ ] Other: ___________________

### Refresh triggers (review immediately when these happen)

- [ ] Tool UI changes that break a saved prompt
- [ ] New bank policy or procedure that affects an entry
- [ ] New regulator bulletin that touches an entry's domain
- [ ] My role or department changes
- [ ] Compliance officer flags an entry
- [ ] An entry hasn't been used in 6+ months → consider retiring

### Sharing schedule

- [ ] Quarterly share with my manager
- [ ] Annual review with compliance contact
- [ ] Submit to bank's Foundation Champion / L&D when requested

### Retirement criteria

I will retire (delete or archive) an entry if any of these are true:

- Not used in 6 months
- The underlying task has substantially changed
- The tool referenced is no longer approved
- A planned successor entry replaces it
