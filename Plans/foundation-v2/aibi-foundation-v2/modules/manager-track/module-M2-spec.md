# Manager Track — Module M2: Reading and Reviewing a Personal Prompt Library

**Track:** Manager · **Duration:** 30 min
**Activity types used:** 6 (find the flaw — review 3 sample team libraries)
**Daily-use outcomes:** Library Review Worksheet (per direct report)

---

## Why this module exists

Personal Prompt Libraries are useless to the bank if no one reviews them. The library is a window into how a team member actually uses AI — what they're prompting, what data tier they're touching, what tools they're using, what's working.

Module M2 gives the manager three sample (synthetic) libraries to review. The manager learns to read a library quickly, spot the issues, and produce a coaching-ready review.

---

## Learning objectives

By the end, the manager will:

1. Review a Personal Prompt Library in under 10 minutes
2. Identify common issues: tier mismatches, missing verification, decay, drift
3. Produce a structured review worksheet for each direct report
4. Distinguish constructive feedback from policing

---

## Activities

### Activity M2.1 — Review three sample libraries (20 min · type 6)

**Setup (1 min):** "Three Personal Prompt Libraries from fictional team members. Review each. The platform will reveal what you missed."

**The activity:**

Three synthetic libraries presented (each with 5–8 entries), each with planted issues:

**Library A — The Foundation graduate doing it well.**
- 6 entries, all schema-conformant
- Multi-model tested
- Quarterly review dates filled in
- Issue: subtle drift — one entry's "data tier" says Internal but the sample input contains a member name (small but worth coaching)

**Library B — The over-user.**
- 12 entries, some thinly tested
- Several entries with "Last tested" 90+ days ago (decay)
- One entry with a Restricted data tier listed but using a Public AI tool (catastrophic misalignment)
- Issue: needs urgent intervention on the misalignment; broader coaching on quality vs. quantity

**Library C — The under-user.**
- 3 entries, all from the course modules (no original work)
- Verification fields blank
- "Frequency" all marked "rare"
- Issue: not actually using AI — coaching opportunity to find what's blocking adoption

For each library, the manager:
1. Reads the entries
2. Tags issues by category (tier mismatch / decay / drift / verification gap / quality issue / good)
3. Drafts a one-paragraph coaching note for the team member

After submission, the platform reveals the planted issues and the rubric grading.

**Capture:** the manager's review of each library + the rubric → Library Review Worksheet template.

### Activity M2.2 — Build your worksheet (8 min · type 5)

**Setup (1 min):** "Now you have a worksheet you can use for every direct report."

**The activity:**

Platform pre-fills a Library Review Worksheet template with:
- Reviewer name (the manager)
- Direct report name
- Library version reviewed (date)
- Issues by category
- Coaching note draft
- Follow-up date

Manager customizes for their team and saves.

**Capture:** Library Review Worksheet template → `module_M2_library_review_worksheet.md`.

### Activity M2.3 — "Schedule the first reviews" callout (1 min · reading)

> *"Pick three direct reports. Schedule a 30-minute library review with each in the next two weeks. Use this worksheet. Coach, don't police. The conversation matters more than the worksheet."*

---

## Daily-use outcomes

1. A **Library Review Worksheet** template for each direct report
2. **Internalized review pattern** — 10 minutes per library, structured
3. **A schedule** for the manager's first reviews

---

## Assessment criteria

- All three sample libraries reviewed
- Tier-mismatch issue (Library B) caught
- At least 4 of the planted issues across the three libraries identified
- Worksheet template customized
