# Module 17: Personal Prompt Library

**Pillar:** Application · **Duration:** 30 min · **Track:** Foundation Full
**Activity types used:** 5 (build & test — library stress-tested across models), 8 (real-world capture)
**Daily-use outcomes:** Personal Prompt Library, ≥5 entries, multi-model verified

---

## Why this module exists

This is the spine artifact. Every prior module has produced something. Module 17 promotes those artifacts into structured, durable Personal Prompt Library entries — and stress-tests each entry against multiple models so the learner knows which works where.

The library schema is fixed (defined in `course-guide.md` Section 9). Authors and platform must respect it: the same schema rolls up to the Departmental Skill Library in AiBI-Specialist.

---

## Learning objectives

By the end, the learner will:

1. Curate at least 5 Personal Prompt Library entries from prior module artifacts
2. Stress-test each entry across multiple models
3. Capture model-strength notes per entry
4. Build a maintenance plan for the library

---

## Activities

### Activity 17.1 — Promote prior artifacts to library entries (15 min · type 5)

**Setup (1 min):** "Your prior modules already produced library-ready content. The platform will promote each one into a fully-structured entry."

**The activity:**

Platform shows the learner's accumulated artifacts:
- Module 1 Member Communication
- Module 8 Cheat Sheet starter prompts
- Module 10 Project Brief system prompt
- Module 11 Document Workflow Prompt
- Module 12 Excel/AI Patterns (3)
- Module 14 Workflow Map
- Module 16 Role Use-Case Card

For each, the platform asks: "Promote to library? (Y/N)"

When promoted:
1. Platform pre-fills the schema fields from the source artifact
2. Learner fills in any missing fields (especially Time saved per use, Last tested, Quarterly review)
3. Platform displays the entry in canonical schema form

Target: at least 5 promotions.

**Capture:** library entries in canonical schema.

### Activity 17.2 — Stress-test the library (10 min · type 5)

**Setup (1 min):** "Each library entry should work across models. The platform will test."

**The activity:**

For each promoted entry:
1. Platform sends the entry's system prompt + sample input to all three models (Claude, ChatGPT, Gemini).
2. Side-by-side responses.
3. Platform asks: "Does each model produce acceptable output? Where does each shine? Where does each struggle?"
4. Learner annotates the entry's "Notes / what didn't work" field with model-strength observations.

**Capture:** annotated library with model-strength notes per entry.

### Activity 17.3 — Maintenance plan (4 min · type 1)

**Setup (1 min):** "A library decays without a plan. Build yours."

**The activity:**

Platform shows the maintenance plan template:

- Review cadence (default quarterly)
- Refresh triggers (tool change, policy change, role change, regulator bulletin)
- Sharing schedule (manager review quarterly, compliance annually)
- Retirement criteria (not used in 6 months, task changed, tool no longer approved)

Learner customizes; platform formats.

**Capture:** library with maintenance plan → `module_17_personal_prompt_library.md`.

### Activity 17.4 — "Back at the bank" callout (1 min · reading)

> *"Your Personal Prompt Library is yours. Share it with your manager. When AiBI-Specialist deploys at your bank, your library is the input to the Departmental Skill Library — same schema, rolled up. Don't change field names. The schema is the contract."*

## Daily-use outcomes

1. A **Personal Prompt Library** with ≥5 entries, all multi-model tested
2. **Model-strength notes** per entry
3. **A maintenance plan** that prevents decay

---

## Assessment criteria

- At least 5 entries promoted from prior modules
- All entries conformant to schema (no missing required fields)
- Each entry has multi-model stress-test results
- Maintenance plan filled in with realistic cadence and triggers

---

## Author/facilitator notes

- **Schema is sacred.** Never let learners (or future authors) change field names. Specialist tier authoring depends on this contract.
- **Honest blanks beat fake fills.** A field marked "—" with a note is better than fabricated data.
- **The stress-test is a real differentiator from v1.** Learners walk away knowing not just which prompts work, but which work where.
