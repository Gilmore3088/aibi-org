What is a skill?

    
Most institutions encountering Claude do so through chat — a compliance officer pastes a regulation, a credit analyst types a question, an operations manager drafts a memo. The conversations are useful. But they are also fleeting. Every chat starts from zero. Every prompt is reinvented.

    
A skill is the discipline of writing it down once. It's a single instruction file Claude reads when invoked by a trigger phrase. Saved in your Toolbox, the skill becomes part of how the institution works — auditable, transferable, governed. Used this way, Claude becomes less of a clever assistant and more of an institutional capability.

    
Skills are saved as .md (markdown) files with structured metadata. They can be deployed in any Claude-based interface that supports them — including Anthropic's Cowork desktop product, custom integrations through the API, or your bank's internal AI tools.

  


  

    
Six operating principles.

    
A useful prompt is improvisation. A useful skill is engineering.

    

      
i.
A skill has an owner.
Every skill is assigned to a single role. Unowned skills are abandoned skills.

      
ii.
A skill is versioned.
Track version, created date, last reviewed. When a regulation changes, the version increments.

      
iii.
A skill asks first.
The clarifying questions are non-negotiable. Five questions prevent ninety percent of rework.

      
iv.
A skill never invents citations.
Skills don't reason from general knowledge. They cite documents they've actually read. URLs, guidance numbers, examiner names — if the source isn't provided, the skill says so explicitly rather than fabricating one that looks real.

      
v.
A skill knows when to stop.
Every skill has explicit escalation triggers — conditions to hand off to a human.

      
vi.
A skill is piloted.
No skill enters production on day one. One user runs it for two weeks and revises it.

Recommended folder structure.

    
For institutions deploying skills via Anthropic's Cowork or a similar local-folder integration, this is a starting layout. The same structure works as a GitHub repository for your skills library.

    

/your-bank-skills/

├─ README.md overview, deployment notes

├─ about-me/

│  ├─ about-me.md role, institution, preferences

│  └─ institution.md products, members, branches

├─ skills/ your saved .md skill files

│  ├─ credit-memo.md

│  ├─ exam-prep.md

│  └─ …

├─ policies/ institutional policies

├─ templates/ document & deck templates

├─ data/ CSVs, exports, reference

└─ outputs/ where Claude saves work
