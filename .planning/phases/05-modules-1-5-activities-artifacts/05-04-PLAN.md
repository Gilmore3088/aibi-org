  <name>Task 1: WCAG 2.1 AA accessibility audit and fixes</name>
  <files>src/app/courses/aibi-p/_components/ActivityForm.tsx, src/app/courses/aibi-p/_components/ClassificationDrill.tsx, src/app/courses/aibi-p/_components/SubscriptionInventory.tsx, src/app/courses/aibi-p/_components/AcceptableUseCardForm.tsx</files>
  <action>
Systematic audit of all Phase 5 interactive components against A11Y-01 through A11Y-05.

**A11Y-01 — Keyboard operability:**
For each component, verify and fix:
1. All interactive elements reachable via Tab key in logical order.
2. Radio buttons: Arrow keys navigate within group, Tab moves between groups.
3. Submit buttons: Enter key submits.
4. ClassificationDrill: Add keyboard shortcut hints — "Press 1 for Tier 1, 2 for Tier 2, 3 for Tier 3" displayed visually and announced by screen reader. Wire keydown handler for 1/2/3 keys during active drill.
5. Focus management: after form submit, move focus to the success message or download button (not left on disabled submit).
6. Add visible focus rings on all interactive elements: `focus-visible:ring-2 focus-visible:ring-[color:var(--color-terra)] focus-visible:ring-offset-2`. Use focus-visible (not focus) to avoid showing rings on mouse clicks.
7. Skip link: if not already present in the course layout, add one at the top of the module page that links to the main content area.

**A11Y-02 — Color not sole indicator:**
Audit and fix every place where color conveys meaning:
1. ClassificationDrill timer: already has text labels ("Time remaining: Ns", "Hurry! Ns remaining") per Plan 02. Verify this is implemented.
2. ClassificationDrill results: correct/incorrect must have BOTH icon AND text label. Verify green checkmark has "Correct" text, red X has "Incorrect" text.
3. SubscriptionInventory: radio selections should have visible selected state beyond just the radio button fill color — add a subtle background highlight on the selected option row.
4. Form validation errors: red text alone is not enough — add an error icon (exclamation mark) and/or prefix "Error:" to the message text.
5. Timer progress bar: add aria-valuenow, aria-valuemin, aria-valuemax, and role="progressbar" with aria-label.

**A11Y-03 — Alt text:**
1. Audit all icons used (checkmark, X, clock, arrow) — each must have either aria-label or be wrapped in a span with sr-only text if decorative icons have adjacent visible text.
2. The SVG arrow icon in the "Next Module" link already has aria-hidden="true" — verify and keep.
3. Any images in module content sections need alt text (none expected in M1-5, but verify).

**A11Y-04 — Contrast ratios:**
Verify these specific combinations against 4.5:1 (body) / 3:1 (large text):
1. var(--color-dust) text on var(--color-linen) background — this is used for hint text. Dust is a light gray; check contrast ratio. If below 4.5:1, darken the hint text to at least #666 on #f9f6f0.
2. var(--color-terra) (#b5512e) on white — used for accent text and activity labels. This should pass 3:1 for large text but verify for small text usage.
3. White text on var(--color-terra) (#b5512e) background — used for submit buttons. Verify 4.5:1.
4. Error text var(--color-error) (#9b2226) on white/parchment — should pass.
5. Timer text colors: verify sage and terra on white meet 4.5:1.
6. Fix any failures by adjusting the non-brand color (e.g., darken hint text, not change terra).

**A11Y-05 — Downloads without JS:**
Verify that all artifact download links use plain `<a href="..." download>` tags. The AcceptableUseCardForm's download link should be a regular anchor to the GET endpoint, not a JS-triggered fetch.

Apply all fixes. Add aria-live="polite" regions for:
- Form submission success/error messages
- ClassificationDrill score reveal
- Timer countdown (use aria-live="off" to avoid constant announcements — screen readers read it on demand)
