Client Component for M5 Activity 5.1 — the timed 20-scenario classification drill (M5-04).

Props: { activity: Activity, enrollmentId: string, moduleNumber: number, scenarios: DrillScenario[], existingResponse?: Record&lt;string, unknown&gt; | null, onSubmitSuccess?: (activityId: string) => void }

Where DrillScenario is: { scenario: string, tier: string, reasoning: string } — extracted from the m5-drill-scenarios table rows in module-5.ts.

Create a type for this at the top of the file:
```typescript
interface DrillScenario {
  readonly scenario: string;
  readonly tier: string;
  readonly reasoning: string;
}
```

State machine phases: 'ready' | 'active' | 'review' | 'submitted'

**Ready phase:**
- Show drill introduction: "You will classify 20 banking scenarios. 20 seconds per scenario. Your classification determines which AI tools may be used."
- "Start Drill" button.

**Active phase:**
- Show one scenario at a time. Progress indicator: "Scenario {n} of 20".
- Three radio buttons for Tier 1/2/3 (same options from activity field definition).
- Countdown timer: 20 seconds per scenario displayed as a progress bar that depletes AND a numeric countdown. Timer uses requestAnimationFrame or setInterval at 100ms for smooth countdown. Use var(--color-sage) for plenty of time, var(--color-terra) when under 5 seconds. Also show a text label that changes — NOT color-only (A11Y-02): "Time remaining: {N}s" and when under 5 seconds: "Hurry! {N}s remaining".
- If timer expires without selection: record as "no-answer" (treated as incorrect).
- On selection OR timer expiry: advance to next scenario after a 300ms delay.
- Store answers in an array: { scenarioIndex: number, selected: string | null, correct: string, timeRemaining: number }[]

**Review phase:**
- Show overall score: "{correct}/20 ({percentage}%)" in a large display.
- Tier breakdown: how many correct per tier.
- List all 20 scenarios with:
  - Correct: green checkmark icon + "Correct" text label (not color-only — A11Y-02)
  - Incorrect: red X icon + "Incorrect" text label + the correct answer + one-sentence reasoning from DrillScenario.reasoning
  - Timed out: orange clock icon + "Time expired" text label + correct answer + reasoning
- "Submit Results" button at bottom.

**Submitted phase:**
- POST to /api/courses/submit-activity with response: { answers: [...the answer array], score: N, total: 20, timeElapsed: totalMs }.
- On success: show "Drill submitted" confirmation, call onSubmitSuccess.

**If existingResponse provided:**
- Skip directly to a read-only review showing saved results.

Keyboard: Tab moves between radio options and buttons. Enter/Space selects. Timer does NOT auto-advance if the user is mid-keyboard-navigation — only after explicit selection.

Mobile (390px): Full-width scenario text, stacked radio buttons below, timer bar at top of the drill card.
