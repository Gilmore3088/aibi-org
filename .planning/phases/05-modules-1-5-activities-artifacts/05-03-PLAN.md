**API Route — POST /api/courses/generate-acceptable-use-card:**

1. Authenticate user using cookies() + createServerClient (same pattern as submit-activity).
2. Parse body: { enrollmentId: string } OR accept query param ?enrollmentId= for GET requests.
3. Support both POST (generate from fresh submission) and GET (re-download from saved response):
   - POST body: { enrollmentId, responses: { roleContext, primaryAiTool, highestRiskScenario, quickWinUseCase } }
   - GET: read from activity_responses table for activity_id '5.2' + enrollment_id
4. Verify enrollment ownership (user_id check).
5. For GET: query activity_responses for enrollment_id + activity_id='5.2'. If not found, return 404.
6. Import renderToBuffer from @react-pdf/renderer (server-side).
7. Render AcceptableUseCardDocument with the response data.
8. Return the PDF buffer with headers:
   - Content-Type: application/pdf
   - Content-Disposition: attachment; filename="AiBI-Acceptable-Use-Card.pdf"

Handle errors: 401 (not authenticated), 403 (not enrolled / not owner), 404 (no activity response found for GET), 500 (PDF generation failed).
