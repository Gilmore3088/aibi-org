1. **MarkdownRenderer.tsx**: Server Component. Props: `{ content: string, className?: string }`.
   - Renders Markdown strings to HTML. Use a lightweight approach: split by `\n\n` for paragraphs, handle `**bold**`, `*italic*`, `- ` list items, and `### ` headings.
   - Do NOT add a Markdown library (remark/rehype) -- the content is simple enough for regex-based rendering. Keep it under 80 lines.
   - Wrap in `<div className="prose">` with Tailwind prose-like styles: `text-sm leading-relaxed text-[color:var(--color-ink)]`, paragraphs with `mb-4`, bold with `font-semibold`, lists with `list-disc pl-5`.
   - If content contains backtick code, render in `<code>` with DM Mono font.

2. **ModuleHeader.tsx**: Server Component. Props: `{ moduleNumber: number, title: string, pillar: Pillar, estimatedMinutes: number, keyOutput: string }`.
   - Full-width header band with pillar color background: `bg-[color:${PILLAR_META[pillar].colorVar}]` with linen text
   - Per SHELL-11: header bands are color-coded by pillar (Sage/Awareness, Cobalt/Understanding, Amber/Creation, Terra/Application)
   - Module number in DM Mono: "Module 0N" format
   - Title in Cormorant serif, large (text-4xl lg:text-5xl), italic
   - Below title: estimated time and key output in DM Mono small caps
   - Padding: px-8 py-12 lg:py-16
   - Pattern from mockup: accent line before module number, institutional styling

3. **ContentSection.tsx**: Server Component. Props: `{ section: Section }`.
   - Section title in Cormorant serif (text-2xl font-bold)
   - Content rendered via MarkdownRenderer
   - If section has subsections, render them nested with smaller heading size
   - Generous bottom margin (mb-16) between sections
   - Section id as HTML id attribute for anchor linking

4. **ContentTable.tsx**: Server Component. Props: `{ table: ContentTable }`.
   - Caption above table in Cormorant serif
   - Table with proper thead/tbody, thead has parchment-dark background
   - Column headers in DM Mono uppercase small text (text-[10px] tracking-widest)
   - Rows with hover state (hover:bg-[color:var(--color-parch)])
   - Cell text in body font, proper padding
   - Horizontal scroll wrapper on mobile: `overflow-x-auto` with `-webkit-overflow-scrolling: touch`
   - Per SHELL-09: table must not cause horizontal page scroll on 390px

5. **ActivityFormShell.tsx**: Client Component ('use client' -- needs form state for field display). Props: `{ activity: Activity }`.
   - Renders the activity title and description
   - For each field in `activity.fields`:
     - `text`: `<input type="text">` with placeholder
     - `textarea`: `<textarea>` with placeholder and rows=4
     - `radio`: radio button group from field.options
     - `select`: `<select>` dropdown from field.options
     - `file`: `<input type="file">` (disabled, shell only)
   - Show minLength requirement text below fields: "Minimum N characters"
   - Submit button present but disabled with text "Activity submission available when enrolled"
   - Styled as a bordered card with left border in pillar color: `border-l-4 border-[color:var(--color-terra)]`
   - This is a SHELL only -- no form submission logic. Activity interactivity is Phase 5-6 per CONTEXT.md deferred items.

  <files>src/app/courses/aibi-p/[module]/page.tsx</files>
  <action>
1. **[module]/page.tsx**: Server Component. This is the dynamic route for individual modules.

   - `generateStaticParams()`: Return `[{ module: '1' }, { module: '2' }, ..., { module: '9' }]` from the modules array. This pre-renders all 9 module pages at build time.

   - `generateMetadata({ params })`: Return `{ title: 'Module N: ${title} | AiBI-P' }` using getModuleByNumber.

   - Page component receives `params: { module: string }`:
     - Parse module number from params: `const moduleNum = parseInt(params.module, 10)`
     - Look up module: `const mod = getModuleByNumber(moduleNum)`
     - If not found or moduleNum < 1 or > 9: `notFound()` from next/navigation
     - Render in order:
       1. `<ModuleHeader>` with module data and pillar color
       2. Content sections: map over `mod.sections` rendering `<ContentSection>` for each
       3. Tables: map over `mod.tables` (if any) rendering `<ContentTable>` for each, placed after relevant sections based on content flow
       4. Activities: map over `mod.activities` rendering `<ActivityFormShell>` for each at the bottom
     - Wrap in `<article className="max-w-4xl mx-auto px-6 lg:px-8 py-8">` for content width
     - Add "Next Module" link at bottom (unless module 9): link to `/courses/aibi-p/${moduleNum + 1}` with terra-colored button
     - Add "Back to Overview" link: link to `/courses/aibi-p`

   - For the table/section interleaving: some modules have tables that belong contextually between sections. A simple approach: render all sections first, then all tables, then all activities. This matches the content structure where sections contain the prose, tables are supplementary data, and activities are at the end.

   - No enrollment check in this plan (Plan 03 handles SHELL-12). The page renders for everyone temporarily.
