**2. Create save-progress API route (src/app/api/courses/save-progress/route.ts):**
POST handler accepting { enrollmentId: string, moduleNumber: number }.

Server-side forward-only enforcement (SHELL-05):
- Authenticate user via supabase.auth.getUser(). Return 401 if absent.
- Read enrollment: SELECT id, user_id, completed_modules, current_module FROM course_enrollments WHERE id = enrollmentId AND user_id = user.id. Return 403 if mismatch.
- Validate: moduleNumber must equal current_module (cannot skip). Return 400 if moduleNumber != enrollment.current_module.
- Validate: all prior modules (1..moduleNumber-1) must be in completed_modules (use canAccessModule from courseProgress.ts logic inline — do not import server module in API route; duplicate the 3-line check). Return 400 "prior modules incomplete".
- Write: UPDATE course_enrollments SET completed_modules = completed_modules || [moduleNumber], current_module = moduleNumber + 1 WHERE id = enrollmentId. Use service role client.
- If moduleNumber is 9 (last module), set current_module to 9 (stay at last, do not set to 10).
- Return 200 { success: true, nextModule: moduleNumber + 1 }.
