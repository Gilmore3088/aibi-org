### 4. Voice / style decisions documented

All 13 emails follow these rules (locked in across this branch):

- Author-direct opening, no big H1 wall
- Plain paragraphs + bordered card menus
- Conversational reply-back ask before signature
- DM Mono numbers, percentages stay as `X%` (brand rule wins over
  Every's "X percent")
- `<strong>` reserved for visual labels ‚Äî italics for prose emphasis
- Em dashes carry no surrounding spaces (`X‚ÄîY`, never `X ‚Äî Y`)
- Subject lines = headline (title case OK); H1 inside body = subhead
  (sentence case, no terminal period)
- No semicolons in email copy
- Sourced statistics only (Gartner, FDIC, AIEOG, Personetics, Apiture)
- No "AI-powered" buzzwords
- Tier-specific reply-back asks designed to provoke a real answer

## Open production issue (NOT fixed in this branch)

**Symptom:** purchasing the In-Depth Assessment routes the buyer to
`https://www.aibankinginstitute.com/assessment/in-depth?reason=no-purchase`
after Stripe payment instead of `/assessment/in-depth/take`.

**Reported by:** James Gilmore (paid as `jlgilmore2+e2e1@gmail.com`)
**Reported at:** end of this session, not yet investigated

**Root-cause candidates** (in order of likelihood):

1. Stripe webhook didn't fire in production ‚Äî `STRIPE_WEBHOOK_SECRET`
   mismatch in Vercel, missing webhook endpoint registration, or
   webhook errored silently. Check Vercel function logs for
   `/api/webhooks/stripe` from time of purchase.
2. Email mismatch ‚Äî buyer paid Stripe with email A, logged into
   Supabase with email B. The `/take` page lookup at
   `src/app/assessment/in-depth/take/page.tsx:64-73` matches on
   `user_id` OR `email`, so a stale auth account would miss.
3. `provisionEnrollment` ran but errored ‚Äî same Vercel logs would show.

**Diagnostic plan** (post-merge):

- Check Supabase `course_enrollments` for any row with
  `email = 'jlgilmore2+e2e1@gmail.com'` AND
  `product = 'in-depth-assessment'`
- If absent ‚Üí webhook never wrote it ‚Üí check Vercel logs and
  Stripe webhook delivery dashboard
- If present but `/take` still redirects ‚Üí email mismatch; check
  `auth.users.email` for the same address
