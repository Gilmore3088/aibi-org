01 · Recommended Architecture

The recommended stack
and why.

Option A — Recommended
Next.js 14 (App Router) + Vercel

Frontend framework + hosting. The HTML prototype (aibi-site-v3.html) converts cleanly to React components. Vercel handles CI/CD, edge functions, and analytics integration. Free tier sufficient for launch.

Why: The assessment tool requires client-side state management. Next.js handles SSR for SEO on marketing pages and CSR for the interactive assessment. Vercel deployment is a git push.
Option B — No-Code Alternative
Webflow + Memberstack + Typeform

If timeline is critical and developer capacity is limited: Webflow for the marketing site, Typeform for the assessment tool (with redirect to results page), Memberstack for future course access gating.

Tradeoff: Less control over assessment UX, harder to iterate on scoring logic, Typeform branding visible. Acceptable for MVP if timeline pressure is real.
Full Recommended Stack
Frontend: Next.js 14 (App Router), TypeScript, Tailwind CSS (utility classes match the prototype approach)
Hosting: Vercel — free tier for MVP, Pro ($20/mo) when traffic warrants
Email: ConvertKit API for list management + automated sequences
CRM: HubSpot (free tier) — contact creation via API on form submissions
Scheduling: Calendly embed (standard iframe or popup widget)
Payments: Stripe — Checkout Sessions for $97 and $295 products
Analytics: Plausible Analytics (privacy-friendly, simple script tag, GDPR-compliant)
Font loading: Google Fonts (Cormorant, Cormorant SC, DM Sans, DM Mono) — already specified in prototype
Future — LMS: Kajabi (external) or custom-built with Supabase + Postgres if budget allows

03 · Assessment — Technical Spec

The most important
feature to build correctly.

Build Priority
The assessment is the primary conversion mechanism. It must work perfectly on mobile (iPhone Safari, Android Chrome) in under 3 minutes. One question per view on mobile. Progress bar visible throughout. Score animation on results. This gets the most QA time before launch.
State Management
Use React useState for all assessment state. No server calls during question flow — questions and options are hardcoded, not fetched.

State shape: 
interface AssessmentState {
  currentQuestion: number;        // 0–7
  answers: number[];              // Array of scores (1–4) per question
  phase: 'questions' | 'score' | 'results';
  email: string;                  // Captured after score reveal
  emailCaptured: boolean;
}

// Score tiers
const getTier = (total: number) => {
  if (total >= 28) return { label: 'Ready to Scale', color: '#4a6741' };
  if (total >= 22) return { label: 'Building Momentum', color: '#b5512e' };
  if (total >= 15) return { label: 'Early Stage', color: '#c96a43' };
  return { label: 'Starting Point', color: '#9b2226' };
};
Email Capture Flow
When phase transitions to 'score':
1. Animate score ring into view
2. Show tier label and description
3. Show email capture form ("Get your personalized AI roadmap")
4. On email submit: POST to /api/capture-email → ConvertKit API + HubSpot API
5. Transition phase to 'results' → show dimension breakdown + CTAs

The email form must NOT be required to see the score. Phase 'score' shows the number. Phase 'results' shows the breakdown — that's what's behind the email gate. 
// /api/capture-email route
// POST body: { email, score, tier, answers[] }
export async function POST(req: Request) {
  const { email, score, tier, answers } = await req.json();

  // 1. Add to ConvertKit with tags
  await fetch('https://api.convertkit.com/v3/forms/[FORM_ID]/subscribe', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      api_key: process.env.CONVERTKIT_API_KEY,
      email,
      tags: [`score-${tier.replace(' ', '-').toLowerCase()}`],
      fields: { score: String(score), tier }
    })
  });

  // 2. Create HubSpot contact
  await fetch('https://api.hubapi.com/crm/v3/objects/contacts', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${process.env.HUBSPOT_API_KEY}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      properties: {
        email,
        assessment_score: score,
        score_tier: tier,
        lead_source: 'assessment'
      }
    })
  });

  return Response.json({ success: true });
}
⚠ Assessment State Persistence — Browser Back / Refresh
The assessment uses React useState which resets on page refresh or browser back navigation. On mobile, where browsers aggressively kill tabs, users can lose their progress mid-assessment.

Fix: Sync state to sessionStorage on every answer. Restore on mount. Clear on email capture (signifies completion). This is a one-time useEffect pattern — add it during Phase 1 before launch. 
// Persist and restore assessment state
useEffect(() => {
  const saved = sessionStorage.getItem('aibi-assessment');
  if (saved) {
    const state = JSON.parse(saved);
    setAnswers(state.answers);
    setCurrentQuestion(state.currentQuestion);
  }
}, []);

useEffect(() => {
  if (answers.length > 0) {
    sessionStorage.setItem('aibi-assessment', JSON.stringify({
      answers, currentQuestion
    }));
  }
}, [answers, currentQuestion]);

// Clear on email capture:
sessionStorage.removeItem('aibi-assessment');
Score Ring Animation
SVG circle with strokeDasharray and strokeDashoffset technique. Animate offset from full (score = 0) to calculated value (score = total/32) over 1.5s with cubic-bezier(0.4, 0, 0.2, 1) easing. Use useEffect to trigger animation after component mounts. Colors map to tier: Starting Point → #9b2226, Early Stage → #b5512e, Building Momentum → #c96a43, Ready to Scale → #4a6741.
04 · ROI Calculator Component

Client-side calculation.
No server required.

// ROICalculator.tsx — all math client-side
interface ROIInputs {
  fte: number;        // Total FTEs (default: 85)
  costPerFTE: number; // Loaded annual cost (default: 58000)
  loHours: number;    // Conservative hrs/FTE/week automatable (default: 1.5)
  hiHours: number;    // Optimistic hrs/FTE/week automatable (default: 3)
}

const calcROI = ({ fte, costPerFTE, loHours, hiHours }: ROIInputs) => {
  const hourlyRate = costPerFTE / 2080;
  const midHours = (loHours + hiHours) / 2;

  return {
    mid: fte * midHours * hourlyRate * 50,
    low: fte * loHours * hourlyRate * 50,
    high: fte * hiHours * hourlyRate * 50,
    hoursPerYear: Math.round(fte * midHours * 50),
    efficiencyPoints: ((fte * midHours * hourlyRate * 50) / (fte * costPerFTE) * 100).toFixed(1)
  };
};

// Format helper
const fmt = (n: number): string => {
  if (n >= 1_000_000) return `$${(n / 1_000_000).toFixed(2)}M`;
  if (n >= 1_000) return `$${Math.round(n / 1000)}K`;
  return `$${Math.round(n)}`;
};
