## Per-surface findings

### Onboarding flow — `WelcomeFirstPrompt` → `OnboardingSurvey`

**Score impact:** Pillars 1, 4, 5, 6 all positive.

- The composition is genuinely clean and the copy is on-brand. The "That is the floor. Every module from here gives you one of these" line lands as a promise without a buzzword.
- **Brittle path: streaming-fetch failure UX is competent but unstyled.** If `/api/sandbox/chat` returns non-200 the banker sees `errorMessage` in `text-[color:var(--color-error)]` mono inside the same parchment-bordered section, plus a Skip prompt. This is correct, but the error string from the catch is generic ("Could not reach the AI sandbox. Try again in a moment."). No retry button. A 1-line addition: a "Try again" `<button onClick={runPrompt}>` next to the skip would close that loop.
- **Concern: the demo prompt is sent to the live Anthropic sandbox before the banker has interacted with anything else.** This is a real cost line per fresh enrollment. Not a UI fix, but worth flagging — the rate-limit map is keyed on `moduleId: 'aibi-p-welcome'` (line 54) which is correct. Confirm the per-key budget at the API layer is conservative for the welcome-key.
- **No visible `lang` issue or contrast issue.** Eyebrow is terra `font-serif-sc text-xs tracking-[0.22em]` — matches the convention from the audit baseline.

### Per-module Apply forms — `ActivityForm` + `module-activities.ts`

**Score impact:** Pillars 1, 4, 5, 6 all positive.

- **Field labels match the artifact promise.** Sampled M1 (`email-rewrite.md`), M3 (`prompt-card.md`), M4 (`about-me.md`), M5 (`project-brief.md`), M7 (`tool-score.md`). Each form's field set maps 1:1 to the template's `{{placeholders}}`, so the artifact renders coherently and the banker can predict what the output will look like.
- **The `[field_id not provided]` fallback in the merge** (`generate-module-artifact/route.ts:89-95`) is the right call — a visible "not provided" reads as honest, where a silent `{{tone}}` would read as broken. Consider one polish: replace `field_id` with the human-readable label from the spec, e.g., `[Tone you prefer in AI output — not provided]`. Trivial, ships with the next pass.
- **Artifact download CTA flow.** After submit, the success region focuses, the read-only restating of the response renders in `bg-[color:var(--color-parch)]`, and a "Your artifact is ready" mono eyebrow + outlined download anchor sits below a divider. The download anchor uses `currentColor` for the icon and outline-style hover (terra fills on hover). Right register — feels like collecting a receipt, not a marketing claim.
- **One small inconsistency.** `ActivityForm.tsx:476-477` — the visible button label is `Download {activity.artifactId?.replace(/-/g, ' ').replace(/\b\w/g, (c) => c.toUpperCase())}`. For `aibi-p-m1-email-rewrite` this yields "Aibi P M1 Email Rewrite". Cosmetic, but reads slightly off. Suggest hard-coding the visible label per spec (`spec.title` or a dedicated `downloadCtaLabel`) so M1's button reads "Download email rewrite (.md)".
- **Form pattern repeats `placeholder:text-[color:var(--color-dust)]`** — correct dust usage now, AA-fine because placeholder is by definition non-essential.

### Resend email template — `src/lib/resend/index.ts`

**Score impact:** Pillar 4 positive; Pillar 6 positive.

- **Email-client compatibility is correct.** `<table cellpadding="0" cellspacing="0" border="0">` nesting matches the Litmus-recommended boilerplate. All styles inline. Hex values not CSS variables. Max-width 560 with center alignment. Tested by inspection — should render in Outlook 2016+, Gmail web, iOS Mail, Apple Mail without surprises.
- **Tone match.** "No marketing list, no sales follow-up unless you ask. The free assessment gives you the diagnostic; everything beyond is on your timeline." This is the same voice as the home Trust strip and the tier interpretations. Preserve.
- **One typography note.** Email body uses `font-family:Georgia,serif` for headings and `font-family:'DM Sans',Arial,sans-serif` for body, `font-family:'DM Mono',Consolas,monospace` for the score. Most email clients won't have DM Sans / DM Mono installed, so they fall back to Arial / Consolas — which is correct (hosted webfonts in email are fragile). The Georgia-as-Cormorant-fallback is a defensible substitution and arguably more readable in mail than a webfont download anyway.
- **`escape()` is applied to every payload-derived string** (lines 122, 137, 156, 159, 167). No XSS surface. `padEnd(28, ' ')` in the text version may truncate label visually if a dimension label exceeds 28 chars but won't break — minor.
- **Reply-to is `hello@aibankinginstitute.com`** (hardcoded, line 24). If that mailbox isn't yet routed, every reply silently bounces. Confirm the mailbox is live before the first send. Not a code change, an ops check.

Three of them under a `font-mono tabular-nums` treatment with named citations would lift Pillar 4 to 4/4 without buzzword cost. Prior audit's exact recommendation; still the highest-leverage Pillar 4 move.
